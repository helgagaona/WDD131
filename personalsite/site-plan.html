Overview

Purpose

The purpose of my website is to provide visitors with a daily dose of inspiration and motivation through a fun and interactive quiz. By taking the quiz, users can select what inspires them most—whether it's motivation, creativity, or peace—and receive a personalized quote of the day based on their choice. The website aims to create an engaging and uplifting experience, offering visitors a moment of reflection and positivity as they go about their day. It’s a place to explore different perspectives and get inspired, all while enjoying a simple, user-friendly experience.

Audience

The target audience includes individuals seeking daily inspiration, people interested in self-improvement, busy individuals who want quick motivational content, quiz enthusiasts, and followers of motivational or positive thinking content. The website is designed to be accessible and engaging for users of all ages who appreciate reflective quotes and daily positivity.

Dynamic Elements

  • Interactive quiz: Users will fill out a quiz to select their mood. JavaScript functions like `getQuizData()` and `displayQuote()` will be used to process the quiz and display a personalized quote. The form will listen for the `submit` event and update the page dynamically.
  • Filtering: Users will be able to filter quotes by author and category. JavaScript will use the `filter()` method to display quotes based on selected criteria, and DOM manipulation will update the page with matching quotes.
  • Forms: Users will be able to submit their own quotes. The form will listen for the `submit` event, and the new quote will be saved to Firebase using an `addQuote()` function, showing how objects and arrays will be used to store and manage quotes.
  • JavaScript logic: Conditional statements (`if/else`) will be used to display different quotes based on the user's mood selection, such as showing an inspirational quote if "happy" is chosen.
  • DOM Manipulation: JavaScript will interact with the DOM to select elements and update them, such as showing the quiz results and updating the quote list when the user filters quotes.
  • Event listeners: JavaScript will listen for events like form submissions (`submit`) and filter changes to trigger actions like displaying personalized quotes and filtering the quote list.
  • Objects: Quotes will be stored as objects with properties like `author`, `text`, and `category`. These will be manipulated to display quotes based on user input.
  • ES Modules: JavaScript will be organized into modules (e.g., `quiz.js` for the quiz logic, `quotes.js` for managing quotes), helping to keep the code modular and maintainable.
  • Arrays and Array Methods: Arrays will be used to store quotes, and methods like `filter()`, `map()`, and `forEach()` will be used to manage and display quotes dynamically. For example, `filter()` will be used to show quotes by category, and `map()` could be used to create an array of authors.
